## Reporting a Vulnerability

SmartForm is a web platform that uses Firebase Firestore as its cloud
database. Security is a top priority.  
If you discover any vulnerability related to:

- Firestore security rules  
- Unauthorized access to surveys or responses  
- Data manipulation or privilege escalation  
- Exposure of sensitive information  
- Authentication or permission bypass  
- Injection or malformed data attacks  

Please report it immediately.

### How to report

Send an email or message to the SmartForm security team:

**Email:** `smartform.security@proyecto.com`  
*(replace with a real contact if applicable)*

### What to expect

After submitting a report:

1. You will receive an acknowledgment within **48 hours**.  
2. The issue will be reviewed and reproduced in a controlled environment.  
3. If valid, the vulnerability will be rated (high/medium/low).  
4. A fix will be deployed within **24 hours to 7 days**, depending on severity.  
5. You will be notified once the issue is resolved.

### If the report is declined

If a vulnerability cannot be reproduced or does not pose a risk:

- You will receive a detailed explanation.  
- You may request a follow-up review.

## Security Commitment

SmartForm commits to:

- Maintaining safe and updated Firebase security rules  
- Protecting user survey data and responses  
- Preventing unauthorized access or modifications  
- Following best practices in UI validation and input handling  
- Reviewing all reports seriously and promptly  
- Constantly improving the security of the platform
